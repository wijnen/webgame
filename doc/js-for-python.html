JavaScript Cheat Sheet for Python Programmers

Feature Python JavaScript Explanation
Command Terminators 
parrot = 'dead'
				
parrot = 'dead';
Because JavaScript doesn't recognize line endings as command terminators, every command must be terminated by a semicolon.
Code Blocks 
if cheese == 'cheddar':
	print('out of stock')
	return
while life_has_meaning():
	pass
				
if (cheese == 'cheddar') {
	print('out of stock');
	return;
}
while (life_has_meaning()) {}

Technically, JavaScript allows exactly one statement after an if, while, etc. Instead of a normal statement, it is possible to use a compound statement. This is a seried of statements enclosed in curly braces. The JavaScript interpreter makes no distinction between newline and space, so indentation is irrelevant. However, incorrect indentation makes for unmaintainable code, so this "feature" should never be used.

There is no pass statement in JavaScript; an empty compound statement should be used instead. Note that an empty regular statement (a semicolon) is also allowed. This is a common mistake and it should therefore never be done on purpose.
Special Constants 
None
None
				
null
undefined
JavaScript has two values similar to Python's None value. null is a value that can be assigned and conventionally means that the value is absent, similar to how None is conventionally used. undefined can also be assigned, but is mostly the result of things that have not be assigned, such as the return value of a function that didn't provide one, or the value of an object member that isn't defined.
Operators 
found(item) and item == spoon
found(item) or item == spoon
not found(item)
item is spoon
item is not spoon
10 ** 35
				
found(item) && item == spoon
found(item) || item == spoon
!found(item)
item === spoon
item !== spoon
Math.pow(10, 35)
Other operators are identical.
Debugging Output 
print('Get on with it!')
				
console.info('Get on with it!');
The output that is produced can be viewed in the debugging console (ctrl-shift-k in Firefox). If you use console.error instead, it'll include a stack backtrace.
Creating Variables 
name = 'Brian'
age = 25
				
var name = 'Brian';
var age;
age = 25;
In JavaScript, a variable must be defined before it can be used.
Conditional Expressions 
if name == 'Brian':
	gender = 'Not a girl'
while age < 25:
       age += 1
				
if (name == 'Brian')
	gender = 'Not a girl';
while (age < 25)
	age += 1;
The conditional expression must be in parenthesis. It is not followed by a colon.
For Loops on Lists 
for i, cheese in enumerate(cheeses):
	print(cheese)
				
for (var i = 0; i < cheeses.length; ++i) {
	var cheese = cheeses[i];
	console.info(cheese);
}
It's best to treat this for statement as a magic line that just needs to be written this way. var may be omitted if i is defined previously.
For Loops on Objects 
for type in cheeses:
	print(type, cheeses[type])
				
for (var type in cheeses)
	console.info(type, cheeses[type]);
Lists 
cheeses = ['cheddar', 'gouda', 'parmezan']
print(cheeses[5])	// raises an exception.
last = cheeses.pop()
cheeses.append('mozarella')
first = cheeses.pop(0)
				
var cheeses = ['cheddar', 'gouda', 'parmezan'];
cheeses.note = 'out of stock';
console.info(cheeses[5]); // prints "undefined".
var last = cheeses.pop();
cheeses.push('mozarella');
var first = cheeses.splice(0, 1)[0];
Lists in JavaScript are called arrays. They are also objects and can have non-numerical items. Tuples don't exist in JavaScript. Undefined items are returned as the special undefined object. Negative indices cannot be used in JavaScript. Push and pop can be used to manipulate the end of the list. Splice can be used to add or remove items from other position. Its first argument is the position to act. The second is the number of items to remove. Those are returned as an array. Any further arguments are placed in the list instead of the removed items (if any).
Lengths 
print(len(cheeses))
cheeses[1:] = []
				
console.info(cheeses.length);
cheeses.length = 1;
The length attribute is implicitly defined for arrays. Arrays can be truncated or expanded by changing its value. They are also automatically expanded by assigning a value to an item larger than the current length.
Dicts 
person = { 'name': 'Brian', 'age': 25 }
print(person['name'])
print(person['age'])
				
var person = { name: 'Brian', 'age': 25 }
console.info(person['name']);
console.info(person.age);
All object keys are forced to be strings in JavaScript. The quotes may be omitted when defining an object if the value is a valid identifier. Items can be referenced either using square brackets, or (if the member name is a valid identifier) as an attribute.
Function Definitions 
def have_argument():
	return "no, it isn't"
				
function have_argument() {
	return "no, it isn't";
}
var have_argument = function() {
	return "no, it isn't";
};

A function is defined with the function keyword. The body is inclosed in curly braces. Note that it is not a compound statement: the braces cannot be omitted if there is only one statement.

The function keyword can only be used as a command at top level. Inside a function body or other statement, a function expression must be assigned to a variable instead. Note that there is a semicolon after the function expression.
Function Expressions 
def romans():
	return 'Go Home!'
paint(times = 100, text = romans)
				
paint(100, function() {
	return 'Go Home!';
});
(Older) JavaScript does not have a short notation for lambda functions. However, it does have a syntax for writing regular functions as an expression. So while Python makes the lambda function more readable, JavaScript makes callback functions more usable by placing them in the function call instead of defining them before it. Note that the entire function expression is within the parenthesis of the function call. Function expression do not need to have a name.
Function Arguments 
def paint(times, text = 'Romans They Go The House'):
	print(text)
paint(100, 'Romans, Go Home!')
paint(times = 100)
paint(text = 'Romans, Go Home!', times = 100)
def paint(*args):
	print(args[0]);
				
function paint(times, text) {
	if (text === undefined)
		text = 'Romans They Go The House';
	console.info(text)
}
paint(100, 'Romans, Go Home!');
paint(100);
paint(100, 'Romans, Go Home!');
function paint() {
	console.info(arguments[0]);
}
Keyword arguments cannot be used in JavaScript. Giving more or less arguments than are declared in the definition is not an error. Missing arguments are set to undefined. All arguments, including extra arguments, can be accessed through the special variable arguments.
